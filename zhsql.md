# 中文結構化查詢語言
結構化查詢語言（Structured Query Language, SQL）於1974年由Boyce和Chamberlin提出，並實現於國際商用機器公司（IBM）關係數據庫系統SystemR上實現；出於功能豐富、使用方便靈活、語言簡潔易學等優勢，深受計算機工業界和用戶采用。1980年10月，美國國家標準局（ANSI）數據庫委員會X3H2批准SQL作爲數據庫語言標準，同年公佈了標準SQL，爲國際標準化組織（ISO）所采納。
結構化查詢語言功能可分爲3部分：數據定義、數據操縱和數據控制。其核心基於關係代數，又超出關係代數，有聚集、數據庫更新等功能。特點是：
1. 數據描述、操縱、控制等功能一體化。
2. 兩種使用方式，統一的語法結構：
   - 一是聯機交互使用，作爲自含型語言，適於一般人員使用；
   - 二是嵌入高級程序設計語言（如C、Go語言），適於計算機專業人員使用。
3. 高度非過程化。結構化查詢語言是第四代語言（4GL），用戶只需提出「做什麼」，無須具體指明「怎麼做」，存取路徑由系統自行選擇，處理過程由系統自決操作。
4. 語言簡潔，易學易用。結構化查詢語言儘管功能極强，核心功能只用9個動詞。SQL的語法接近英語口語，所以，用戶很容易學習和使用。
然而，現有結構化查詢語言基於英語而非漢語，不便於中國人管理數據庫，而且有時用多個單詞表達一個意思。
